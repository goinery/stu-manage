# 学生选课管理系统 —— 数据库操作指导

## 一、数据库连接信息

当前项目使用的数据库配置如下（位于 `application.properties`）：

```
数据库类型：openGauss（PostgreSQL 兼容）
连接地址：jdbc:postgresql://120.46.181.48:26000/oasys
用户名：dboper
密码：dboper@123
```

你可以使用以下工具连接数据库进行操作：

- **Navicat**（推荐，支持 PostgreSQL）：新建 PostgreSQL 连接，填入上述地址、端口、用户名和密码
- **pgAdmin**：PostgreSQL 官方管理工具
- **DBeaver**：免费开源，支持多种数据库
- **命令行**：`psql -h 120.46.181.48 -p 26000 -U dboper -d oasys`

> 注意：如果需要切换回 MySQL，取消 `application.properties` 中 MySQL 配置的注释，注释掉 PostgreSQL 配置即可。

## 二、数据库表结构总览

系统包含以下核心数据表：

| 表名 | 说明 | 主要字段 |
|------|------|---------|
| `student` | 学生表 | id, studentNumber(学号), username(姓名), password, phone, email, roleId, collegeId, state |
| `teacher` | 教师表 | id, loginName(登录名), username(姓名), password, phone, email, roleId, collegeId, state |
| `manage_user` | 管理员表 | id, loginName, username, password, roleId, state |
| `course` | 课程表 | id, courseName, collegeId, teacherId, classPlace, classDate, startDate, endDate, credits, academicYear, optional, selected |
| `college` | 学院表 | id, name, teacherNum, studentNum, state |
| `menu` | 菜单表 | id, name, parentId, href, sort, state |
| `role` | 角色表 | id, name, state |
| `role_menu_rel` | 角色-菜单关联表 | id, roleId, menuId |
| `student_course_rel` | 学生-课程关联表 | id, studentId, courseId, teacherId, isQualified, state |
| `course_academic_year` | 学年表 | id, academicYear, state |
| `information` | 首页公告表 | id, title, content, publishDate, roleId |

## 三、常用数据库操作指南

### 3.1 添加学生数据

```sql
INSERT INTO student (id, studentNumber, username, password, phone, email, roleId, collegeId, createDate, state)
VALUES (
    '这里填32位UUID',        -- id：主键，使用UUID去掉横线
    '2024001',                -- studentNumber：学号（唯一）
    '张三',                   -- username：学生姓名
    '123456',                 -- password：初始密码
    '13800138000',            -- phone：手机号
    'zhangsan@example.com',   -- email：邮箱
    '3',                      -- roleId：角色ID，学生固定为 '3'
    '1',                      -- collegeId：学院ID，对应 college 表的 id
    NOW(),                    -- createDate：创建时间
    '1'                       -- state：状态，'1'表示正常，'0'表示禁用
);
```

**生成UUID的方法（在数据库中）：**
```sql
-- PostgreSQL / openGauss
SELECT replace(cast(gen_random_uuid() as varchar), '-', '');
```

**示例 - 批量添加学生：**
```sql
INSERT INTO student (id, studentNumber, username, password, phone, email, roleId, collegeId, createDate, state) VALUES
(replace(cast(gen_random_uuid() as varchar),'-',''), '2024002', '李四', '123456', '13800138001', 'lisi@example.com', '3', '1', NOW(), '1'),
(replace(cast(gen_random_uuid() as varchar),'-',''), '2024003', '王五', '123456', '13800138002', 'wangwu@example.com', '3', '2', NOW(), '1');
```

### 3.3 添加课程数据

```sql
INSERT INTO course (id, courseName, collegeId, teacherId, classPlace, classDate, startDate, endDate, credits, academicYear, teachEndDate, optional, primaryAmount, selected, remark)
VALUES (
    replace(cast(gen_random_uuid() as varchar),'-',''),
    '高等数学',              -- courseName：课程名称
    '1',                     -- collegeId：所属学院ID
    '1',                     -- teacherId：授课教师ID（teacher表的id）
    '教学楼A 301',           -- classPlace：上课地点
    '星期一 上午第一节课',    -- classDate：上课时间
    '2026-09-01 00:00:00',   -- startDate：选课开始时间
    '2026-09-15 00:00:00',   -- endDate：选课结束时间
    4.0,                     -- credits：学分
    '2026上半学年',          -- academicYear：学年（需与 course_academic_year 表对应）
    '2026-12-30 00:00:00',   -- teachEndDate：课程结束时间
    60,                      -- optional：可选人数
    0,                       -- primaryAmount：预选人数
    0,                       -- selected：已选人数
    '课程说明内容'           -- remark：课程说明
);
```

> 注意：`academicYear` 字段的值需要在 `course_academic_year` 表中存在。

### 3.4 添加学年数据

```sql
INSERT INTO course_academic_year (id, academicYear, state)
VALUES (
    replace(cast(gen_random_uuid() as varchar),'-',''),
    '2026上半学年',   -- 学年名称
    '0'               -- state：'1'为当前激活学年，'0'为非激活
);
```

> 重要：同一时间只能有一个学年的 state 为 '1'。添加新学年时，需要先将旧学年改为 '0'：
> ```sql
> UPDATE course_academic_year SET state = '0' WHERE state = '1';
> UPDATE course_academic_year SET state = '1' WHERE academicYear = '2026上半学年';
> ```

### 3.6 添加菜单数据

菜单分为一级菜单和二级菜单，通过 `parentId` 关联。`parentId = '0'` 表示一级菜单。

**添加一级菜单：**
```sql
INSERT INTO menu (id, name, parentId, createDate, createId, updateDate, sort, href, state, remark)
VALUES (
    replace(cast(gen_random_uuid() as varchar),'-',''),
    '个人中心',          -- 菜单名称
    '0',                -- parentId：0表示一级菜单
    NOW(),
    '1',                -- createId：创建人ID
    NOW(),
    10,                 -- sort：排序号
    '/user/profile',    -- href：菜单链接
    '1',                -- state：'1'显示
    '个人中心菜单'
);
```

## 四、常用查询操作

### 查看当前所有学生
```sql
SELECT s.studentNumber, s.username, s.phone, s.email, c.name as collegeName, s.state
FROM student s
LEFT JOIN college c ON s.collegeId = c.id
ORDER BY s.createDate DESC;
```

### 查看当前所有教师
```sql
SELECT t.loginName, t.username, t.phone, t.email, c.name as collegeName, t.state
FROM teacher t
LEFT JOIN college c ON t.collegeId = c.id
ORDER BY t.createDate DESC;
```

### 查看课程及其授课教师
```sql
SELECT c.courseName, t.username as teacherName, co.name as collegeName, c.credits, c.academicYear, c.optional, c.selected
FROM course c
LEFT JOIN teacher t ON c.teacherId = t.id
LEFT JOIN college co ON c.collegeId = co.id;
```

### 查看学生选课情况
```sql
SELECT s.username as studentName, c.courseName, c.credits,
       CASE WHEN sc.state = '1' THEN '已评定' ELSE '未评定' END as scoreState,
       CASE WHEN sc.isQualified = '1' THEN '合格' WHEN sc.isQualified = '0' THEN '不合格' ELSE '未评分' END as qualified
FROM student_course_rel sc
LEFT JOIN student s ON sc.studentId = s.id
LEFT JOIN course c ON sc.courseId = c.id;
```

### 查看角色与菜单的关联
```sql
SELECT r.name as roleName, m.name as menuName, m.href
FROM role_menu_rel rmr
LEFT JOIN role r ON rmr.roleId = r.id
LEFT JOIN menu m ON rmr.menuId = m.id
ORDER BY r.name, m.sort;
```

## 五、常用更新操作

### 重置用户密码
```sql
-- 重置学生密码
UPDATE student SET password = '123456' WHERE studentNumber = '学生学号';

-- 重置教师密码
UPDATE teacher SET password = '123456' WHERE loginName = '教师登录名';

-- 重置管理员密码
UPDATE manage_user SET password = '123456' WHERE loginName = '管理员登录名';
```

### 禁用/启用用户
```sql
-- 禁用学生
UPDATE student SET state = '0' WHERE id = '学生ID';

-- 启用学生
UPDATE student SET state = '1' WHERE id = '学生ID';

-- 同理，教师和学生的操作一致
UPDATE teacher SET state = '0' WHERE id = '教师ID';
UPDATE teacher SET state = '1' WHERE id = '教师ID';
```

### 修改学年（切换当前学期）
```sql
-- 先取消当前激活的学年
UPDATE course_academic_year SET state = '0' WHERE state = '1';

-- 激活新的学年
UPDATE course_academic_year SET state = '1' WHERE academicYear = '目标学年名称';
```

## 六、注意事项

1. **ID 生成**：本系统使用 32 位 UUID（无横线）作为主键，在数据库中插入时请使用 `replace(cast(gen_random_uuid() as varchar),'-','')` 生成
2. **state 字段含义**：大多数表中 `state = '1'` 表示正常/启用/显示，`state = '0'` 表示禁用/隐藏
3. **roleId 对应关系**：`'1'` = 管理员，`'2'` = 教师，`'3'` = 学生
4. **学年一致性**：课程的 `academicYear` 字段需要与 `course_academic_year` 表中存在的学年名称一致
5. **选课人数**：添加课程时 `selected` 初始设为 0，学生选课后系统会自动增加
6. **外键关系**：本系统没有使用数据库级别的外键约束，但逻辑上的关联关系需要手动维护（如删除学院前，应先处理该学院下的学生和教师数据）
7. **密码存储**：当前系统密码为明文存储，仅适用于学习和演示环境
